# Forbidden Comment Patterns

## 즉시 삭제 대상

| 패턴 | 이유 |
|---|---|
| 코드 그대로 설명 | 코드가 이미 설명하고 있음 |
| 주석 처리된 코드 | git history로 복원 가능, 혼란만 유발 |
| "이 부분 중요", "주의" 같은 막연한 경고 | 왜 중요한지 없으면 의미 없음 |
| 마침표로 끝나는 주석 | 팀 규칙 위반 |

## 개선 판단 기준

아래 중 하나라도 해당하면 주석 불필요:
- 메서드 이름이 의도를 충분히 설명한다
- 파라미터 타입/이름이 의미를 전달한다
- 바로 위아래 코드로 흐름이 읽힌다

아래 중 하나라도 해당하면 주석 필요:
- 왜 이 방식인지 코드에서 보이지 않는다
- 비즈니스 정책/도메인 제약이 숨어 있다
- 성능 최적화 의도가 코드만 봐선 모른다
- 외부 시스템 특성 때문에 비직관적인 선택을 했다

## TODO 기준

작성해도 되는 TODO:
```kotlin
// TODO: 트래픽 증가 시 집계 테이블 전환 검토
// TODO: User 엔티티 도입 후 resolver 등록
```

작성하면 안 되는 TODO:
```kotlin
// TODO: 나중에 수정
// TODO: 고쳐야 함
```
→ 조건이나 후속 맥락 없는 TODO는 기술 부채만 쌓음
